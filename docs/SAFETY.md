# 安全治理（SafetyGovernanceAgent）

## 总原则

Hermes 是古籍知识系统，不是医疗器械：

1. **患者端绝不提供**：诊断、处方、剂量推荐、停换药建议、治疗替代方案。
   `SafetyGovernanceAgent.check_patient_query` 以意图模式拦截
   （开方/吃什么药/剂量/怎么治/停药/换方…），拦截后给出可做事项清单
   （术语解释、症状日记、就诊准备）。
2. **红旗症状即刻转诊**：胸痛不缓解、呼吸困难、高热不退、意识改变、便血/
   呕血、急性剧痛、严重过敏、抽搐 → 直接输出「请立即就医或拨打当地急救电话」，
   不再进行任何知识问答。
3. **医师端是辅助不是决策**：方证匹配/病案回源/鉴别输出全部附
   「仅供执业医师参考，处方决策须四诊合参自行判断」免责声明，且每个候选
   附证据链与发布级别，不确定点显式列出。
4. **企业文案合规**：`check_marketing_copy` 拦截「根治/治愈率/包治」等
   夸大疗效表述。
5. **可追溯性即安全**：所有输出强制携带证据链（书/章/条文/原文片段），
   Skill 输出强制 safety_notice 字段——无法回源的内容不会被输出。

## 测试保障

`tests/test_safety_and_workbench.py` 固化以下行为：

- 处方/剂量请求 → 拒绝；
- 红旗症状 → urgent + 急救提示；
- 患者解释输出含免责声明、不含处方语言；
- 医师匹配输出含执业判断声明与证据链。

任何修改若破坏上述行为，CI 即失败。
